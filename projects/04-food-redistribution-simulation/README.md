# Food Redistribution Simulation for a University Campus

## Project Overview

This project uses Discrete Event Simulation to analyse and improve a university campus food redistribution process. The aim is to explore how surplus food can move more efficiently through collection, sorting, storage, and redistribution activities.

## Business Problem

Food service operations can generate edible surplus, but redistribution processes may be limited by timing, capacity, queues, coordination, and resource constraints. Operational inefficiencies can increase avoidable food waste.

## Objectives

- Map the food redistribution process.
- Represent the process through simulation.
- Identify bottlenecks and capacity constraints.
- Test alternative operational scenarios.
- Evaluate opportunities to reduce waste and improve process flow.

## Simulation Approach

The model represents key activities in the redistribution process, including:

- surplus food generation;
- collection;
- transport or movement;
- sorting;
- temporary storage;
- redistribution.

Discrete Event Simulation allows process behaviour to be studied over time, including queues, waiting times, resource utilisation, and bottlenecks.

## Scenario Analysis

Alternative process configurations can be compared before changes are made in the real operation. Example questions include:

- What happens if collection frequency increases?
- Where do queues form?
- Is storage capacity sufficient?
- Which activity is the main bottleneck?
- How does additional capacity affect waste and throughput?

## Business Value

Simulation provides a low-risk environment for testing operational changes. It can support decisions around resource allocation, collection scheduling, storage capacity, and process redesign.

## Tools and Techniques

Discrete Event Simulation, process modelling, bottleneck analysis, capacity analysis, scenario testing, operational research, and process improvement.

## Skills Demonstrated

Process mapping, systems thinking, simulation modelling, capacity planning, bottleneck identification, scenario comparison, and evidence-based operational recommendations.

## Future Improvements

The model could be expanded using live demand data, stochastic travel times, cost analysis, carbon-impact measures, volunteer availability constraints, and optimisation of collection schedules.
